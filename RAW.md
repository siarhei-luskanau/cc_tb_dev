Telegram-бот с локальной LLM

🎯 Цель
Разработать Telegram-бота, который работает как простой AI-чат, используя локальную языковую модель (LLM).

🧠 Описание задачи
Необходимо создать Telegram-бота, который:
    - принимает текстовые сообщения от пользователя
    - передает их в локальную LLM
    - возвращает сгенерированный ответ обратно в Telegram
    - ❗Важно: Каждое сообщение должно обрабатываться **независимо**, без хранения истории диалога.


⚙️ Технические требования

1. LLM (обязательно локально)
Использовать Ollama

Рекомендуемые модели:
    - qwen3:0.6b
    - qwen3.5:0.8b

2. Язык программирования
На выбор: Python или JavaScript/TypeScript

3. Telegram API
Использовать Telegram Bot API через:
    - polling
    - НЕ webhook

4. Архитектура
- Минимальная архитектура: `Telegram → Bot → LLM → Bot → Telegram`
- Без базы данных и без хранения состояния.

📦 Функциональные требования

Бот должен:
    - ✅ отвечать на любые текстовые сообщения
    - ✅ корректно обрабатывать ошибки (например, если LLM недоступна)
    - ✅ работать стабильно при нескольких запросах подряд

🚫 Ограничения
    - ❌ Без хранения истории диалога
    - ❌ Без использования облачных LLM (OpenAI, Anthropic и т.д.)
    - ❌ Без базы данных

⭐ Бонус (необязательно)
Можно добавить:
    - обработку start команды
    - настройку системного промпта
    - выбор модели через команду
    - логирование запросов

Предоставить:
    - Исходный код проекта
    - Инструкцию по запуску (README.md)

Минимальный pipeline:
    - Получить сообщение из Telegram
    - Отправить текст в локальную LLM
    - Получить ответ
    - Вернуть ответ пользователю
